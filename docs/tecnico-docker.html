¿Por qué los contenedores puente necesitan re2_net?

Spark y Airflow necesitan conectarse a re2_postgres (host interno del Proyecto 1) para leer las tablas fuente via JDBC. Metabase se conecta a re2_nginx para el reindex de ElasticSearch. Neo4J y Hive solo consumen datos que ya están en Hive, por eso solo están en ra_net.

Imágenes Custom (Dockerfiles)
deployments/Dockerfile.hive
Base: apache/hive:3.1.3 — agrega driver JDBC de PostgreSQL
# Descarga el JDBC de PostgreSQL en el classpath de Hive RUN curl -fsSL https://repo1.maven.org/maven2/org/postgresql/postgresql/42.7.3/postgresql-42.7.3.jar \ -o /opt/hive/lib/postgresql-42.7.3.jar
deployments/Dockerfile.airflow
Base: apache/airflow:2.9 — agrega providers y JDBC para Spark
# Providers de Airflow: Spark, Postgres RUN pip install apache-airflow-providers-apache-spark \ apache-airflow-providers-postgres # JDBC pre-descargado — evita fallas de Ivy/Maven en runtime RUN mkdir -p /opt/airflow/jars && \ curl -fsSL .../postgresql-42.7.3.jar \ -o /opt/airflow/jars/postgresql-42.7.3.jar
Comandos de Verificación
# Ver todos los contenedores del stack P2 docker compose -f deployments/docker-compose.yml --env-file .env ps # Ver redes y verificar re2_net existe docker network ls | grep re2 # Ver volúmenes creados docker volume ls | grep ra_ # Logs de un servicio específico docker compose -f deployments/docker-compose.yml --env-file .env logs -f airflow # Verificar que analytics-db tiene las 3 bases docker exec ra_analytics_db psql -U analytics -d hive_metastore -c "\l" # Verificar que el warehouse Hive tiene datos docker exec ra_hive_server beeline -u "jdbc:hive2://localhost:10000" \ -n root --silent=true \ -e "SELECT COUNT(*) FROM restaurants_dw.fact_items_pedido;" 2>&1 | grep -v SLF4J
