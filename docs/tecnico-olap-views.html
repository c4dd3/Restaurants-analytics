📊 Referencia OLAP — Vistas SQL

Las 6 vistas OLAP del esquema restaurants_dw, con SQL completo y explicación de window functions.

6 vistas Window Functions Metabase + Beeline
Resumen de Vistas
Todas las vistas son CREATE OR REPLACE VIEW — no almacenan datos, consultan las tablas ORC en tiempo real.
Se usan en los dashboards de Metabase (conexión vía SparkSQL / HiveServer2) y también en análisis ad-hoc con Beeline.

Vista 1 — ingresos_por_mes_categoria
1
olap_ingresos_por_mes_categoria
Ingresos, unidades y ticket promedio agregados por mes y categoría de producto. Filtra solo pedidos confirmed.
GROUP BY SUM · COUNT · AVG Dashboard: Ventas
Columnas de salida
anio mes nombre_mes categoria total_pedidos unidades_vendidas ingresos_totales ticket_promedio
CREATE OR REPLACE VIEW olap_ingresos_por_mes_categoria AS SELECT t.anio, t.mes, t.nombre_mes, p.categoria, COUNT(DISTINCT f.pedido_id) AS total_pedidos, SUM(f.cantidad) AS unidades_vendidas, ROUND(SUM(f.monto_total), 2) AS ingresos_totales, ROUND(AVG(f.monto_total), 2) AS ticket_promedio FROM fact_items_pedido f JOIN dim_tiempo t ON f.tiempo_key = t.tiempo_key JOIN dim_producto p ON f.producto_key = p.producto_key WHERE f.estado_pedido = 'confirmed' GROUP BY t.anio, t.mes, t.nombre_mes, p.categoria;
Tablas: fact_items_pedido dim_tiempo dim_producto
Vista 2 — actividad_usuarios_por_restaurante
2
olap_actividad_usuarios_por_restaurante
Usuarios únicos, pedidos e ingresos por restaurante y mes. El restaurante actúa como proxy de zona geográfica.
COUNT DISTINCT Dashboard: Operaciones
Columnas de salida
restaurante zona anio mes nombre_mes usuarios_unicos total_pedidos ingresos
CREATE OR REPLACE VIEW olap_actividad_usuarios_por_restaurante AS SELECT r.nombre AS restaurante, r.direccion AS zona, t.anio, t.mes, t.nombre_mes, COUNT(DISTINCT f.usuario_key) AS usuarios_unicos, COUNT(DISTINCT f.pedido_id) AS total_pedidos, ROUND(SUM(f.monto_total), 2) AS ingresos FROM fact_items_pedido f JOIN dim_tiempo t ON f.tiempo_key = t.tiempo_key JOIN dim_restaurante r ON f.restaurante_key = r.restaurante_key WHERE f.estado_pedido = 'confirmed' GROUP BY r.nombre, r.direccion, t.anio, t.mes, t.nombre_mes;
Tablas: fact_items_pedido dim_tiempo dim_restaurante
Vista 3 — estado_pedidos
3
olap_estado_pedidos
Distribución de pedidos por estado (pending / confirmed / cancelled) con porcentaje relativo dentro del mismo mes.
SUM() OVER PARTITION Dashboard: Ventas
Columnas de salida
anio mes nombre_mes estado_pedido total_pedidos monto_total porcentaje
CREATE OR REPLACE VIEW olap_estado_pedidos AS SELECT t.anio, t.mes, t.nombre_mes, f.estado_pedido, COUNT(DISTINCT f.pedido_id) AS total_pedidos, ROUND(SUM(f.monto_total), 2) AS monto_total, ROUND( COUNT(DISTINCT f.pedido_id) * 100.0 / SUM(COUNT(DISTINCT f.pedido_id)) OVER (PARTITION BY t.anio, t.mes), 2 ) AS porcentaje FROM fact_items_pedido f JOIN dim_tiempo t ON f.tiempo_key = t.tiempo_key GROUP BY t.anio, t.mes, t.nombre_mes, f.estado_pedido;
Window function: SUM(COUNT(DISTINCT pedido_id)) OVER (PARTITION BY anio, mes)
Calcula el total de pedidos del mismo mes sumando los COUNT ya agrupados por estado. Esto permite dividir cada estado entre el total del mes para obtener el porcentaje sin necesidad de un subquery adicional.
Tablas: fact_items_pedido dim_tiempo
Vista 4 — tendencias_consumo
4
olap_tendencias_consumo
Ranking de productos por unidades vendidas dentro de cada categoría y mes. Permite identificar el producto #1 de cada categoría.
RANK() OVER PARTITION Dashboard: Ventas
Columnas de salida
anio mes nombre_mes categoria producto unidades_vendidas ingresos ranking_en_categoria
CREATE OR REPLACE VIEW olap_tendencias_consumo AS SELECT t.anio, t.mes, t.nombre_mes, p.categoria, p.nombre AS producto, SUM(f.cantidad) AS unidades_vendidas, ROUND(SUM(f.monto_total), 2) AS ingresos, RANK() OVER ( PARTITION BY t.anio, t.mes, p.categoria ORDER BY SUM(f.cantidad) DESC ) AS ranking_en_categoria FROM fact_items_pedido f JOIN dim_tiempo t ON f.tiempo_key = t.tiempo_key JOIN dim_producto p ON f.producto_key = p.producto_key WHERE f.estado_pedido = 'confirmed' GROUP BY t.anio, t.mes, t.nombre_mes, p.categoria, p.nombre;
Window function: RANK() OVER (PARTITION BY anio, mes, categoria ORDER BY SUM(cantidad) DESC)
Reinicia el ranking para cada combinación de mes+categoría. El producto más vendido en esa categoría y mes obtiene ranking = 1. A diferencia de ROW_NUMBER(), RANK() asigna el mismo número a empates.
Tablas: fact_items_pedido dim_tiempo dim_producto
Vista 5 — horarios_pico
5
olap_horarios_pico
Volumen de pedidos, unidades e ingresos por hora del día y día de la semana. Distingue entre días de semana y fin de semana.
GROUP BY hora, nombre_dia Dashboard: Operaciones
Columnas de salida
hora nombre_dia es_fin_semana total_pedidos unidades_vendidas ingresos ticket_promedio
CREATE OR REPLACE VIEW olap_horarios_pico AS SELECT t.hora, t.nombre_dia, t.es_fin_semana, COUNT(DISTINCT f.pedido_id) AS total_pedidos, SUM(f.cantidad) AS unidades_vendidas, ROUND(SUM(f.monto_total), 2) AS ingresos, ROUND(AVG(f.monto_total), 2) AS ticket_promedio FROM fact_items_pedido f JOIN dim_tiempo t ON f.tiempo_key = t.tiempo_key WHERE f.estado_pedido = 'confirmed' GROUP BY t.hora, t.nombre_dia, t.es_fin_semana;
Tablas: fact_items_pedido dim_tiempo
Vista 6 — crecimiento_mensual
6
olap_crecimiento_mensual
Crecimiento Month-over-Month (MoM) de ingresos: compara cada mes con el anterior y calcula el porcentaje de cambio.
LAG() OVER ORDER BY Subquery Dashboard: Ventas
Columnas de salida
anio mes nombre_mes ingresos_mes total_pedidos ingresos_mes_anterior crecimiento_pct
CREATE OR REPLACE VIEW olap_crecimiento_mensual AS SELECT anio, mes, nombre_mes, ingresos_mes, total_pedidos, LAG(ingresos_mes) OVER (ORDER BY anio, mes) AS ingresos_mes_anterior, ROUND( (ingresos_mes - LAG(ingresos_mes) OVER (ORDER BY anio, mes)) / NULLIF(LAG(ingresos_mes) OVER (ORDER BY anio, mes), 0) * 100, 2 ) AS crecimiento_pct FROM ( SELECT t.anio, t.mes, t.nombre_mes, ROUND(SUM(f.monto_total), 2) AS ingresos_mes, COUNT(DISTINCT f.pedido_id) AS total_pedidos FROM fact_items_pedido f JOIN dim_tiempo t ON f.tiempo_key = t.tiempo_key WHERE f.estado_pedido = 'confirmed' GROUP BY t.anio, t.mes, t.nombre_mes ) base;
Window function: LAG(ingresos_mes) OVER (ORDER BY anio, mes)
Devuelve el valor de ingresos_mes de la fila anterior en el orden cronológico (mes anterior). El primer mes retorna NULL (no hay mes anterior).

División segura: NULLIF(..., 0) evita división por cero si el mes anterior tuvo ingresos = 0. El subquery base pre-agrega por mes antes de aplicar la window function, ya que LAG no puede usarse directamente sobre agregaciones.
Tablas: fact_items_pedido dim_tiempo
Cómo ejecutar las vistas
Las vistas se crean durante el make setup. Para consultarlas manualmente:
# Desde Beeline docker exec -it ra_hive_server beeline -u jdbc:hive2://localhost:10000 USE restaurants_dw; SELECT * FROM olap_crecimiento_mensual ORDER BY anio, mes; SELECT * FROM olap_tendencias_consumo WHERE ranking_en_categoria = 1; SELECT * FROM olap_horarios_pico ORDER BY total_pedidos DESC LIMIT 10;
