¿Qué es Apache Airflow?

🎯 Airflow es un orquestador

No procesa datos por sí mismo. Es el "director de orquesta" que decide qué se ejecuta, cuándo y en qué orden. Él llama a Spark, a Postgres, al API del Proyecto 1 — pero ellos son los que hacen el trabajo pesado.

📋 Un DAG es el "plan de trabajo"

DAG = Directed Acyclic Graph. Es un archivo Python que describe las tareas y sus dependencias. "Directed" porque las tareas tienen un orden. "Acyclic" porque no hay ciclos — no podés volver a una tarea que ya corrió.

⚙️ Las Tasks son las unidades de trabajo

Cada bloque del DAG es una tarea con un tipo específico: PythonOperator corre código Python, SparkSubmitOperator envía un job a Spark, BranchPythonOperator decide qué camino tomar.

🕐 Schedule = ejecución automática

El DAG está programado con cron 0 2 * * *. Esto significa que Airflow lo lanza solo, todos los días a las 2:00 AM, sin que nadie tenga que hacer nada manualmente.

Descripción de cada tarea
# Tarea Tipo Qué hace Depende de
1 verificar_fuentes Python Hace un SELECT COUNT(*) FROM orders en el Postgres del P1. Si no hay conexión, el DAG falla antes de intentar correr Spark. —
2 cargar_dimensiones Spark Ejecuta spark/jobs/cargar_dimensiones.py. Lee users, restaurants y products del P1, genera llaves surrogate y sobreescribe las 4 tablas de dimensión en Hive. T1
3a cargar_fact_items_pedido Spark Ejecuta spark/jobs/cargar_fact_items_pedido.py. Join de orders + order_items con las dims. Escribe fact_items_pedido en Hive. T2
3b cargar_fact_reservaciones Spark Ejecuta spark/jobs/cargar_fact_reservaciones.py. Carga reservations del P1 a fact_reservaciones en Hive. Corre en paralelo con T3a. T2
4 verificar_cambios_productos Branch Calcula el hash MD5 del catálogo de productos actual y lo compara con el de la ejecución anterior (guardado en Variable de Airflow). Si cambió → rama reindexar. Si no → rama skip. T3a + T3b
5a reindexar_elasticsearch Python Llama a POST /search/reindex del search-service del Proyecto 1 con el token de admin. Solo corre si el catálogo cambió. T4 (rama "cambió")
5b sin_cambios Empty No hace nada. Es el placeholder de la rama "sin cambios" del BranchPythonOperator. T4 (rama "igual")
6 pipeline_completo Empty Marca el fin del pipeline. Usa NONE_FAILED_MIN_ONE_SUCCESS como trigger rule para correr aunque una rama del branch haya sido skipped. T5a o T5b
Conexiones y Variables requeridas en Airflow UI
Conn: postgres_proyecto1
Conexión tipo Postgres al contenedor re2_postgres:5432, base restaurants. Usada por T1 y T4.
Conn: spark_default
Conexión tipo Spark a spark://spark-master:7077. Usada por T2, T3a y T3b.
Var: api_admin_token
JWT de usuario admin del Proyecto 1. Necesario para que T5a pueda llamar a POST /search/reindex.
Var: products_catalog_hash
Generada automáticamente por T4 en la primera ejecución. Guarda el MD5 del catálogo para comparar en la siguiente corrida.
