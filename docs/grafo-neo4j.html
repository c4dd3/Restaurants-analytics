Relaciones
(User)-[:PLACED]→(Order)
Un usuario realizó un pedido. Sin propiedades adicionales.
(Order)-[:CONTAINS]→(Product)
Un pedido incluye un producto.
quantity: INT line_total: FLOAT
(Product)-[:BOUGHT_TOGETHER]→(Product)
Dos productos aparecieron juntos en al menos un pedido. Calculada por Spark como co-ocurrencia.
times: INT (co-ocurrencias)
(User)-[:RECOMMENDS]→(User)
Relación de influencia entre usuarios. Generada heurísticamente por Spark basándose en actividad compartida.
(Location)-[:ROUTE_TO]→(Location)
Ruta de entrega entre dos ubicaciones. Permite encontrar el camino más corto con algoritmos de grafo.
distance_km: FLOAT estimated_minutes: FLOAT
(Courier)-[:ASSIGNED_TO]→(Order)
Un repartidor fue asignado a un pedido. Asignación calculada por Spark con heurística de zona.
Consultas de análisis (02_analysis_queries.cypher)
1. Productos más comprados juntos
Muestra qué productos co-ocurren más veces en pedidos. Útil para recomendaciones y combos.
MATCH (a:Product)-[r:BOUGHT_TOGETHER]->(b:Product) RETURN a.name, b.name, r.times ORDER BY r.times DESC LIMIT 5
2. Usuarios influyentes
Identifica qué usuarios recomiendan a más personas en la red social del sistema.
MATCH (u:User)-[:RECOMMENDS]->(r:User) RETURN u.name, count(r) AS recomendados ORDER BY recomendados DESC
3. Usuarios más activos
Top 10 de usuarios con mayor cantidad de pedidos realizados.
MATCH (u:User)-[:PLACED]->(o:Order) RETURN u.name, count(o) AS pedidos ORDER BY pedidos DESC LIMIT 10
4. Categorías más vendidas
Ingresos y unidades totales por categoría de producto, calculados desde el grafo.
MATCH (:Order)-[r:CONTAINS]->(p:Product) RETURN p.category, sum(r.quantity) AS unidades, round(sum(r.line_total), 2) AS ingresos ORDER BY ingresos DESC
5. Ruta de entrega más corta desde el centro
Encuentra los destinos con menor distancia acumulada desde la ubicación central, recorriendo hasta 8 saltos en el grafo de rutas.
MATCH path = (:Location {location_id: 'loc-central'})-[:ROUTE_TO*1..8]->(dest:Location) WITH path, dest, reduce(total = 0.0, rel IN relationships(path) | total + coalesce(rel.distance_km, 0.0)) AS distancia_km, reduce(tiempo = 0.0, rel IN relationships(path) | tiempo + coalesce(rel.estimated_minutes, 0.0)) AS tiempo_min RETURN dest.name, [node IN nodes(path) | node.name] AS ruta, round(distancia_km, 3) AS distancia_km, round(tiempo_min, 2) AS tiempo_min ORDER BY distancia_km ASC LIMIT 5
Cómo se carga el grafo
1
Spark genera los CSVs
El job de Spark lee Postgres y MongoDB del Proyecto 1, calcula co-compras, recomendaciones y asignaciones de repartidores, y escribe los CSVs en el volumen compartido con Neo4J.
spark/jobs/restaurants_spark_analytics.py --source postgres
2
Se crean los constraints e índices
Define unicidad en los IDs de cada tipo de nodo para evitar duplicados al recargar.
neo4j/queries/00_constraints.cypher
3
Se cargan nodos y relaciones
Carga los CSVs usando LOAD CSV y MERGE. Crea todos los nodos (User, Order, Product, Location, Courier) y sus relaciones.
neo4j/queries/01_load_graph.cypher
4
Se ejecutan las consultas de análisis
Valida la carga y produce los resultados de co-compras, usuarios influyentes, categorías y rutas de entrega.
neo4j/queries/02_analysis_queries.cypher

Para recargar manualmente: make neo4j-load · Para correr las consultas: make neo4j-analysis
