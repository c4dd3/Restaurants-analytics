🗄️ Referencia DDL — Hive Data Warehouse

Definición completa de tablas, columnas, tipos y almacenamiento del esquema restaurants_dw.

Apache Hive 3 ORC + SNAPPY 4 dimensiones · 2 hechos
Contexto
Base de datos: restaurants_dw — creada automáticamente por el script hive/schema/01_dimensions.hql.
Todas las tablas usan ORC con compresión SNAPPY: formato columnar óptimo para consultas analíticas con alto ratio de compresión.
Las llaves surrogate son generadas por Spark usando CRC32 sobre el UUID del Proyecto 1 (sin dependencia de secuencias en Hive).
Los jobs Spark escriben en modo overwrite con FileOutputCommitter v2 para evitar conflictos de rename en HDFS local.
Leyenda: PK Llave surrogate (generada por Spark) FK Llave foránea a dimensión DEG Atributo degenerado del hecho MED Medida / métrica
Tablas de Dimensión
dim_tiempo
Dimensión de tiempo — generada desde timestamps de orders y reservations
Filas estimadas ~8.760 / año
ColumnaTipoRolDescripción
tiempo_key BIGINT PK Surrogate key — formato YYYYMMDDhh (ej. 2024010214 = 2 jan 2024 14:00)
fecha DATE Fecha sin hora (YYYY-MM-DD)
anio INT Año (ej. 2024)
trimestre INT 1 – 4
mes INT 1 – 12
nombre_mes STRING Enero, Febrero … Diciembre (expresión Spark nativa, no UDF)
semana INT Semana del año (1 – 53, ISO)
dia INT Día del mes (1 – 31)
dia_semana INT 1 = Lunes … 7 = Domingo
nombre_dia STRING Lunes, Martes … Domingo
hora INT 0 – 23
es_fin_semana BOOLEAN TRUE si dia_semana ∈ {6, 7}
Almacenamiento: ORC  Compresión: SNAPPY  Script: cargar_dimensiones.py → build_dim_tiempo()
dim_usuario
Usuarios del Proyecto 1 — excluye datos sensibles (password, email completo)
Fuente tabla users (P1)
ColumnaTipoRolDescripción
usuario_key BIGINT PK CRC32(usuario_id) — único por UUID
usuario_id STRING UUID original del Proyecto 1 (para trazabilidad)
nombre STRING Nombre completo del usuario
rol STRING client | admin
fecha_registro DATE Fecha de creación de la cuenta
Almacenamiento: ORC  Compresión: SNAPPY  Script: cargar_dimensiones.py → build_dim_usuario()
dim_restaurante
Restaurantes registrados en el Proyecto 1
Fuente tabla restaurants (P1)
ColumnaTipoRolDescripción

nombre STRING Nombre del restaurante
direccion STRING Dirección física (usada como proxy de zona geográfica)

Almacenamiento: ORC  Compresión: SNAPPY  Script: cargar_dimensiones.py → build_dim_restaurante()
dim_producto
Productos de menú — precio actual desnormalizado (snapshot al momento del ETL)
Fuente tabla products (P1)
ColumnaTipoRolDescripción
producto_key BIGINT PK CRC32(producto_id)
producto_id STRING UUID original del Proyecto 1

Almacenamiento: ORC  Compresión: SNAPPY  Script: cargar_dimensiones.py → build_dim_producto()
Tablas de Hechos
fact_items_pedido
Grano: una fila por ítem dentro de un pedido — tabla de hechos principal
Fuente order_items (P1)
ColumnaTipoRolDescripción
tiempo_key BIGINT FK → dim_tiempo.tiempo_key (hora del pedido)
usuario_key BIGINT FK → dim_usuario.usuario_key
restaurante_key BIGINT FK → dim_restaurante.restaurante_key
producto_key BIGINT FK → dim_producto.producto_key

monto_total DECIMAL(10,2) MED cantidad × precio_unitario — precalculado por Spark
estado_pedido STRING DEG pending | confirmed | cancelled
es_para_llevar BOOLEAN DEG TRUE si el pedido es para llevar
Almacenamiento: ORC  Compresión: SNAPPY  Script: cargar_fact_items_pedido.py

DDL Completo — Crear esquema desde cero
Los 3 scripts HQL se ejecutan en orden. El Makefile los aplica via Beeline dentro del contenedor ra_hive_server como parte de make setup.
01_dimensions.hql
CREATE DATABASE IF NOT EXISTS restaurants_dw COMMENT 'Data Warehouse — Proyecto 2 BD2 TEC'; USE restaurants_dw; CREATE TABLE IF NOT EXISTS dim_tiempo ( tiempo_key BIGINT COMMENT 'Surrogate: YYYYMMDDhh', fecha DATE, anio INT, trimestre INT, mes INT, nombre_mes STRING, semana INT, dia INT, dia_semana INT COMMENT '1=Lunes … 7=Domingo', nombre_dia STRING, hora INT, es_fin_semana BOOLEAN ) STORED AS ORC TBLPROPERTIES ('orc.compress'='SNAPPY'); CREATE TABLE IF NOT EXISTS dim_usuario ( usuario_key BIGINT, usuario_id STRING, nombre STRING, rol STRING, fecha_registro DATE ) STORED AS ORC TBLPROPERTIES ('orc.compress'='SNAPPY'); CREATE TABLE IF NOT EXISTS dim_restaurante ( restaurante_key BIGINT, restaurante_id STRING, nombre STRING, direccion STRING, capacidad INT ) STORED AS ORC TBLPROPERTIES ('orc.compress'='SNAPPY'); CREATE TABLE IF NOT EXISTS dim_producto ( producto_key BIGINT, producto_id STRING, nombre STRING, categoria STRING, precio_actual DECIMAL(10,2), disponible BOOLEAN, restaurante_id STRING ) STORED AS ORC TBLPROPERTIES ('orc.compress'='SNAPPY');
02_facts.hql
CREATE TABLE IF NOT EXISTS fact_items_pedido ( tiempo_key BIGINT, usuario_key BIGINT, restaurante_key BIGINT, producto_key BIGINT, pedido_id STRING, item_id STRING, cantidad INT, precio_unitario DECIMAL(10,2), monto_total DECIMAL(10,2), estado_pedido STRING, es_para_llevar BOOLEAN ) STORED AS ORC TBLPROPERTIES ('orc.compress'='SNAPPY'); CREATE TABLE IF NOT EXISTS fact_reservaciones ( tiempo_key BIGINT, usuario_key BIGINT, restaurante_key BIGINT, reservacion_id STRING, tamano_grupo INT, estado STRING ) STORED AS ORC TBLPROPERTIES ('orc.compress'='SNAPPY');
Verificar desde Beeline
-- Conectarse docker exec -it ra_hive_server beeline -u jdbc:hive2://localhost:10000 -- Listar tablas SHOW DATABASES; USE restaurants_dw; SHOW TABLES; -- Contar filas post-ETL SELECT 'dim_tiempo', COUNT(*) FROM dim_tiempo UNION ALL SELECT 'dim_usuario', COUNT(*) FROM dim_usuario UNION ALL SELECT 'fact_items_pedido', COUNT(*) FROM fact_items_pedido;
